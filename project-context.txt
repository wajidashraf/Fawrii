FAWRII WEBSITE PROJECT CONTEXT

Purpose
- This file stores the main working context, design decisions, implementation patterns, and recent project history for the Fawrii website so future work can continue consistently.

Project overview
- The website is being refined into a polished, professional, tutor-first education platform.
- Main priorities have been:
  - clean and maintainable implementation
  - consistent full-width layout across pages
  - strong visual hierarchy
  - compact, elegant spacing
  - responsive design across devices
  - better cross-browser consistency
  - consistent theme and component styling

Core layout decisions
- Full-width section standard:
  - Main content sections have been standardized to the widest site width.
  - Current standard is max-w-[90rem] / 7.5xl.
- This width is considered final for now unless changed intentionally later.
- Shared section spacing was tightened to make pages feel more premium and less loose.

Theme and design system decisions
- The site theme was aligned to the Fawrii branding extracted from docs/test.html.
- Key brand directions:
  - primary brand color uses teal tones
  - accent color uses orange
  - surfaces and borders were refined for softer but clearer contrast
  - shadows were refined to feel subtle and elegant rather than heavy
- CTA buttons were intentionally reduced in size by about 10 percent from earlier versions.
- The visual goal is:
  - compact
  - elegant
  - modern
  - professional
  - high trust

Typography and hierarchy decisions
- Headings should feel strong and editorial, but not oversized.
- Supporting copy should stay readable with controlled line length.
- Sections now commonly use this hierarchy:
  - section heading
  - one bold key statement or transition sentence
  - supporting muted body text
  - bullets or supporting points inside a callout card where appropriate
  - emphasized closing takeaway
  - CTA

Global styling direction
- Card and panel styling across the site was refined with:
  - subtle border
  - soft shadow at rest
  - slightly stronger shadow on hover
  - smooth transition
- Video embeds and image blocks were also updated to fit the same visual language.
- Cross-browser CSS resets were added for stability, including improvements for Safari and mobile behavior.
- Mobile touch targets were increased where needed.

Important UI patterns currently in use

1. Section widths
- Prefer shared wrappers and keep sections aligned to max-w-[90rem].

2. Card style
- Rounded corners
- Light border
- Soft shadow-sm base
- hover:shadow-md
- Smooth transition-shadow or transition-all
- Use surface/background-alt depending on contrast needs

3. Section content pattern
- For informational sections, the preferred structure is:
  - heading
  - bold intro or key statement
  - callout card or structured list
  - emphasized closing line
  - CTA or supporting link

4. Bullet list dot color
- Bullet dots across home page lists were standardized to orange accent.
- Use bg-accent for bullet dots for consistency.
- If new list dots are added, they should match this standard unless there is a deliberate reason not to.

5. Trusted/signal pills
- Short trust indicators can be presented as rounded inline pills instead of traditional stacked lists when that better suits the content.

6. Flow cards
- Left-to-right process visuals are used where they help communicate progression.
- Reference pattern originally came from the pay-and-rules page platform handling/deductions flow.
- Similar step-flow visuals were requested for:
  - tutoring-on-fawrii page: Apply > Review > Onboard
  - safeguarding page: Issue raised > Review > Resolution

28. Framer Motion animation system
- Installed framer-motion as a dependency.
- Created central animation variants in src/lib/motion.ts:
  - fadeInUp, fadeIn, fadeInLeft, fadeInRight, scaleIn variants
  - staggerContainer, staggerContainerWide, staggerItem for orchestrated reveals
  - Shared easeOut cubic bezier for premium feel
  - viewportConfig (once: true, margin: -80px) for scroll-triggered animations
- Created reusable "use client" motion wrapper components in src/components/ui/motion/:
  - MotionDiv: generic animated div with viewport-triggered entrance, accepts variants and delay
  - StaggerChildren: orchestrates staggered entrance of child elements
  - AnimatedCard: card with stagger entrance + subtle hover lift (y: -4)
  - Barrel export via index.ts
- Architecture: Server components import these client wrappers to add animation without losing SSR/SEO benefits.
- All animations use only transform and opacity (GPU-accelerated, no layout shift).
- All viewport animations fire once (viewport.once = true).
- Framer Motion's built-in prefers-reduced-motion support is active.
- To revert: uninstall framer-motion, delete src/lib/motion.ts and src/components/ui/motion/, remove MotionDiv/StaggerChildren/AnimatedCard imports from all section components.

Practical rules for future edits
- If editing a section, preserve the established visual system first.
- Before introducing a new pattern, check whether one of these existing patterns already solves it:
  - callout card
  - numbered/step card
  - inline trust pills
  - emphasized intro + muted support copy
  - emphasized closing takeaway
  - MotionDiv fadeInUp for section entrance
  - StaggerChildren + AnimatedCard for card grids
  - GradientBlob for decorative depth
  - gradient-cta for CTA section backgrounds
- Keep max width aligned with the site-wide standard.
- Use orange accent for list dots when consistency with current home-page direction is needed.
- Animation wrappers must be "use client" — never add Framer Motion directly to server components.
- Keep animations subtle: use fadeInUp/scaleIn, avoid bounce/spring effects.
- Validate with typecheck after substantive changes.
